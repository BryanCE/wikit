# Task 1.3: Update tsconfig.json

## Purpose

Configure TypeScript to generate type declaration files (.d.ts) for the published package.

## Required Changes

### Enable Type Declarations

```json
{
  "compilerOptions": {
    "declaration": true,
    "declarationMap": true
  }
}
```

- `declaration: true` - Generates .d.ts files alongside .js files
- `declarationMap: true` - Creates .d.ts.map files for better IDE navigation

### Set Output Directories

```json
{
  "compilerOptions": {
    "outDir": "./dist",
    "rootDir": "./src"
  }
}
```

- `outDir` - Where compiled files go
- `rootDir` - Where source files are (maintains structure in dist/)

## Complete tsconfig.json

```json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "ES2022",
    "lib": ["ES2022"],
    "moduleResolution": "bundler",
    "declaration": true,
    "declarationMap": true,
    "outDir": "./dist",
    "rootDir": "./src",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "jsx": "react",
    "resolveJsonModule": true
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules", "dist"]
}
```

## What This Enables

After building with these settings, dist/ will contain:
```
dist/
├── index.js
├── index.d.ts          # TypeScript types
├── index.d.ts.map      # Source mapping
├── commands/
│   ├── index.js
│   ├── index.d.ts
│   └── ...
└── ...
```

## Why This Matters

- TypeScript users get full autocomplete
- IDEs can jump to source definitions
- Type checking works for library consumers
- Better developer experience

## Verification

After updating tsconfig.json:
1. No syntax errors in the JSON
2. Paths match your project structure
3. All necessary compiler options are included
